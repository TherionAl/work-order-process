# 新旧 ERP 数据合并处理规则说明

生成时间：2026年07月15日

适用代码文件：merge_erp_data.py

## 本次调整

程序直接生成原 Excel 后处理中的年度分摊列，不再依赖手工公式追加。

新增字段插入在 `产品金额` 后：

1. 合同天数
2. 去年统计起始日期
3. 去年统计截止日期
4. 去年按期分摊服务费
5. 去年按期分摊服务费（去掉今年倒签的）
6. 今年统计起始日期
7. 今年统计截止日期
8. 今年按期分摊服务费
9. 今年按期分摊服务费（加上倒签去年的服务费）

## 统计区间参数

脚本支持通过命令行参数调整 4 个统计日期：

```powershell
uv run python merge_erp_data.py `
  --last-year-start 2025-01-01 `
  --last-year-end 2025-12-31 `
  --current-year-start 2026-01-01 `
  --current-year-end 2026-12-31
```

参数支持 `YYYY-MM-DD` 和 `YYYYMMDD` 两种格式。

未传参数时，脚本按运行年份自动使用自然年区间。例如 2026 年运行时：

- 去年统计区间：2025-01-01 至 2025-12-31
- 今年统计区间：2026-01-01 至 2026-12-31

## 计算口径

- `合同天数` = `明细运维结束日期 - 明细运维开始开始日期 + 1`。
- 去年/今年按期分摊服务费 = `产品金额 * 统计区间与合同服务期重叠天数 / 合同天数`。
- `去年按期分摊服务费（去掉今年倒签的）`：当 `合同申请年份` 等于今年统计起始日期年份时置为 0，否则保留去年分摊金额。
- `今年按期分摊服务费（加上倒签去年的服务费）`：当 `合同申请年份` 等于今年统计起始日期年份且去年分摊金额大于 0 时，按 `今年分摊金额 + 去年分摊金额` 输出；否则保留今年分摊金额。

旧 ERP 的收入、开票、回款金额分成比例换算逻辑继续保留。
