# 工单数据湖 — 数据库设计说明

> **数据库**：MySQL 8.x（utf8mb4 / InnoDB）
> **默认库名**：`work_order_datalake`
> **本机访问地址**：`172.18.141.13:3306`
> **最后更新**：2026-07-09

---

## 1. 表结构总览

| 表名 | 主键 | 分区 | 说明 |
|---|---|---|---|
| `ticket_detail_main` | `(ticket_id, create_dt)` | 按 `create_dt` 月度 RANGE | 工单核心字段 + 分析维度 |
| `ticket_detail_custom_fields` | `(id, create_dt)` | 按 `create_dt` 月度 RANGE | 工单自定义字段 EAV 明细 |
| `customers` | `customer_id` | — | 客户/公司信息 |
| `contacts` | `contact_id` | — | 联系人信息 |
| `sync_task_log` | `id` (自增) | — | 同步任务执行日志 |
| `personnel` | `employee_no` | — | 本地人员花名册（Excel 导入） |

**设计要点**：
- 两张工单核心表按月分区，便于快速查询与归档；
- 分区表不用外键，靠索引 + 程序逻辑保证关联；
- 高频分析维度冗余到主表顶层，避免每次从明细表聚合；

## 3. 分区策略

`ticket_detail_main` 和 `ticket_detail_custom_fields` 按 `create_dt` 做 **RANGE COLUMNS** 月度分区。

```sql
PARTITION BY RANGE COLUMNS(create_dt) (
  PARTITION p202501 VALUES LESS THAN ('2025-02-01'),
  PARTITION p202502 VALUES LESS THAN ('2025-03-01'),
  -- ... 每月一个分区
  PARTITION p202612 VALUES LESS THAN ('2027-01-01'),
  PARTITION pmax    VALUES LESS THAN (MAXVALUE)   -- 兜底分区
)
```

## 4. 表关系与关联查询

```
ticket_detail_main.cust_user_id        ──→ contacts.contact_id
ticket_detail_main.company_id          ──→ customers.customer_id
contacts.customer_id                   ──→ customers.customer_id
personnel.person_name                  ──→ ticket_detail_main.servicer_user_name  (按姓名，存在重名)
```

>  **注意**：工单系统内的 `servicer_user_id` 是内部用户 ID，与 `personnel.employee_no` 不是同一编号体系，当前不支持直接 ID 关联。

## 7.  SQL 查询场景示范

### 按年月统计工单量

```sql
SELECT create_month_label,
       COUNT(*) AS ticket_count,
       SUM(CASE WHEN ticket_status = '4' THEN 1 ELSE 0 END) AS solved_count,
       SUM(CASE WHEN ticket_status = '5' THEN 1 ELSE 0 END) AS closed_count
FROM ticket_detail_main
WHERE create_month_label = '2026-06'
GROUP BY create_month_label;
```

### 按产品线统计工单分布

```sql
SELECT product_line, COUNT(*) AS cnt
FROM ticket_detail_main
WHERE create_month_label = '2026-06'
  AND product_line IS NOT NULL
GROUP BY product_line
ORDER BY cnt DESC;
```

### 按地域 + 问题类型交叉分析

```sql
SELECT province, problem_type, COUNT(*) AS cnt
FROM ticket_detail_main
WHERE create_month_label = '2026-06'
  AND province IS NOT NULL
GROUP BY province, problem_type
ORDER BY province, cnt DESC;
```

### 按客服姓名关联人员表（注意存在重名）

```sql
SELECT t.ticket_id, t.subject, t.servicer_user_name,
       p.employee_no, p.province, p.role_names, p.group_name
FROM ticket_detail_main AS t
JOIN personnel AS p ON t.servicer_user_name = p.person_name
WHERE t.create_month_label = '2026-06'
LIMIT 100;
```

### 查询某工单的所有自定义字段

```sql
SELECT field_name, field_value, field_value_type
FROM ticket_detail_custom_fields
WHERE ticket_id = 22256891
  AND create_month_label = '2025-06'
ORDER BY field_order;
```

### 按字段名跨工单搜索

```sql
SELECT t.ticket_id, t.subject, c.field_value
FROM ticket_detail_custom_fields AS c
JOIN ticket_detail_main AS t
  ON c.ticket_id = t.ticket_id
  AND c.create_month_label = t.create_month_label
WHERE c.field_name = '产品线'
  AND c.field_value = 'BOSS'
  AND c.create_month_label = '2026-06';
```

### 客户工单统计

```sql
SELECT c.customer_name, c.customer_type, COUNT(*) AS ticket_count
FROM ticket_detail_main AS t
JOIN customers AS c ON t.company_id = c.customer_id
WHERE t.create_month_label = '2026-06'
GROUP BY c.customer_name, c.customer_type
ORDER BY ticket_count DESC
LIMIT 20;
```
